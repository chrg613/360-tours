# Internationalization (i18n)

This document describes the current internationalization state and the recommended approach for future multi-language support.

## Current state

The platform is **English-only**. However, several foundations for internationalization exist:

### Country code support

- `PhoneInput` component with international country code selector.
- `countryCodes.ts` contains a comprehensive list of country codes, dial codes, and flags.
- Phone numbers are stored in E.164 format (e.g., `+91XXXXXXXXXX`).

### Multi-market intent

The business plan targets:
1. **Phase 1**: US real estate photographers.
2. **Phase 2**: India and emerging markets.
3. **Phase 3**: Global expansion.

This implies multi-language support will be needed.

## Recommended approach

### String externalization

- Use a library like `react-i18next` or `react-intl`.
- Extract all user-visible strings into JSON translation files.
- Organize translations by feature/page.

### Translation file structure

```
src/locales/
  en/
    common.json
    tours.json
    auth.json
    dashboard.json
  hi/         # Hindi
    common.json
    ...
```

### Number and date formatting

- Use `Intl.NumberFormat` for numbers (file sizes, view counts).
- Use `Intl.DateTimeFormat` or `date-fns` locale support for dates.
- Current formatting utilities in `src/utils/format.ts` should be made locale-aware.

### RTL layout

- If Arabic or Hebrew are added, ensure CSS supports RTL.
- Use CSS logical properties (`margin-inline-start` instead of `margin-left`).
- Test layout mirroring for all components.

## Priority languages

| Language | Market | Priority |
|----------|--------|----------|
| English | Global | Current |
| Hindi | India | High |
| Spanish | Latin America | Medium |
| Arabic | Middle East | Medium (requires RTL) |

## Implementation phases

1. **Extract strings** — Move all hardcoded strings to translation files.
2. **Add language switcher** — UI for selecting language, persisted to user preferences.
3. **Translate priority languages** — Hindi and Spanish first.
4. **RTL support** — CSS and layout changes for Arabic.
